End-to-End Cloud Data Pipeline

A comprehensive data pipeline processing Solar System data from raw API to analytics-ready datasets

Project Overview

This project demonstrates a complete data engineering workflow, transforming raw Solar System API data into analytics-ready datasets using Google Cloud Platform. The pipeline implements automated data extraction, quality controls, and metric calculations while maintaining development best practices through version control and automated testing.

View Project on GitHub

1. Data Collection & Storage

  • Python script calls Solar System OpenData API
  • API response parsed into planet and moon JSON files
  • BigQuery client loads JSON data into staging tables
  • Raw data preserves original structure and French naming

2. Base Transformations

  • Planet table creation:
    • Generates unique PlanetId
    • Standardizes field names
    • Extracts nested JSON values (mass, volume)
    • Applies consistent units of measurement
  • Moon table creation:
    • Maps moons to parent planets
    • Handles French-to-English planet name translation
    • Creates standardized MoonId system
    • Maintains orbital relationship data

3. Data Quality Framework

  • Automated assertions verify:
    • Physical properties (mass, density, gravity)
    • Orbital parameters (perihelion < aphelion)
    • Valid measurement ranges
    • Required field population
  • Quality checks run before transformation completion
  • Failed validations trigger pipeline alerts

4. Metrics Layer

  • Aggregated moon statistics by planet:
    • Population counts and distributions
    • Physical characteristic averages
    • Size classifications
    • Orbital parameter analysis
  • Handles sparse data appropriately
  • Focuses on well-populated metrics
  • Ready for visualization consumption

5. Orchestration & Scheduling

  • GitHub Actions triggers daily API extract
  • Dataform manages transformation sequence:
    • Raw data validation
    • Base table creation
    • Data quality checks
    • Metrics calculations
  • Error handling and notification system
  • Automated cleanup processes

6. Development Workflow

  • Changes developed in solar-system-dev
  • Testing and validation in development
  • PR review process
  • Deployment to solar-system-prod
  • Daily production refresh via Dataform
